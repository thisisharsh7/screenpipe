## week of april 21, 2026

### new features

- **focus-aware capture** — a new opt-in mode that throttles recording on monitors you're not actively looking at, cutting CPU usage on multi-monitor setups. off by default; enable it in settings.
- **Microsoft 365 & Teams OAuth** — connect Microsoft 365 and Teams with full OAuth 2.0 via Microsoft Graph instead of pasting tokens manually. see [connections](/connections).
- **Google Docs integration** — a new first-class Google Docs connector with OAuth, a settings card showing your recent docs, and proxy routing through your local screenpipe server.
- **QuickBooks Online + Google Sheets connectors** — OAuth 2.0 for both, plus a "cancel" button on the OAuth "connecting…" spinner so you're no longer stuck waiting.
- **Bitrix24 CRM** — new integration for Bitrix24 portals across all regions (.com, .eu, .ru, .de, etc.). configure in settings → connections.
- **browser extension popup** — the extension now opens a popup with a live connection-status dot and quick screen-history search.
- **LAN access for the API** — a new `listenOnLan` toggle under **Privacy → Security** lets the screenpipe API accept requests from other machines on your network. API auth is force-enabled whenever it's on, so an unauthenticated API can never be published to the LAN.
- **Opus 4.7 now available to logged-in users** — Opus 4.7 is open to the logged-in tier (~10 calls/day). Subscribers keep their full quota, and the quota weight was lowered to reflect Opus 4.7's ~3× lower pricing.
- **privacy filter service** — a new CPU-only PII filter service designed to run inside a Tinfoil confidential enclave in front of outbound LLM traffic. [privacy filter docs](/privacy-filter).

### updates

- **summarize meetings with AI** — the meetings → "summarize with AI" flow now reliably opens the Home chat and auto-sends, with a clearer error toast if it can't. see [meeting transcription](/meeting-transcription).
- **apply & restart actually restarts** — toggling auth or pressing "Apply & Restart" in the settings no longer silently no-ops. auth tokens now update on every restart, not just the first.
- **Google Sheets OAuth scope** — added `drive.metadata.readonly` so listing files stops failing with `ACCESS_TOKEN_SCOPE_INSUFFICIENT`. re-connect Sheets to pick it up.
- **connection proxy query params** — the connections proxy now forwards caller query strings, so Sheets `values:append?valueInputOption=USER_ENTERED` and similar calls finally work as expected.

### bug fixes

- **keyword search results** — `/search/keyword?group=false` no longer silently drops results that had no text position. query results now match the grouped endpoint.
- **memories pipeline** — fixed a chain of bugs that caused memory-writing pipes to silently stop creating memories. memories now keep updating, and the UI shows a warning if they go stale.
- **stuck "stopped — click to record"** — if a configured monitor isn't connected, the tray now surfaces the error and retries cleanly instead of silently no-opping.
- **macOS clipboard memory leak** — fixed a long-running memory leak (and occasional crash) in the clipboard capture thread.
- **app crash on startup after upgrade** — fixed a migration crash (`no such table: speaker_embeddings`) that could block boot on some installs when upgrading to recent builds.
- **audio loss on missing speaker model** — if the speaker-segmentation model file is missing or corrupted, audio is no longer dropped. screenpipe now falls back to speaker-unknown transcription and auto-redownloads the model.
- **database pool timeouts** — fixed a connection-starvation issue that caused occasional `pool timed out` errors when writing UI events and audio transcriptions.
- **UTF-8 crash on multibyte characters** — fixed a panic that could occur in the database layer when logging transcriptions containing Cyrillic, emojis, or other multibyte characters.
